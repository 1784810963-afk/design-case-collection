# PRD：AI驱动的设计案例收集应用

## 1. 产品概述

### 1.1 产品简介
一款专为设计师打造的智能案例收集工具。用户只需在编辑器中粘贴设计案例链接，AI将自动提取封面图、概括设计理念并生成提示词标签，帮助用户快速建立个人设计灵感库，告别杂乱无章的收藏夹。

### 1.2 核心价值
- **AI智能分析**：自动提取封面、生成设计理念概括和提示词，省去手动整理的麻烦
- **美观强大的编辑器**：类Notion的富文本编辑体验，支持图文混排，卡片式案例展示
- **高效管理**：搜索、筛选、分类功能，快速找到需要的灵感
- **无缝跳转**：一键访问原网页，保持案例可追溯性

### 1.3 目标用户
- **UI/UX设计师**：收集界面设计案例，按风格、类型整理灵感库
- **平面设计师**：收集海报、品牌、排版案例，提取设计元素
- **产品经理和创意工作者**：收集产品设计参考，分析设计趋势

### 2.3 AI分析功能详解

#### 2.3.1 封面图提取

**提取策略**

优先级顺序：
1. **网站指定封面**：使用网站的og:image或主图
2. **AI视觉评估**：如无指定封面，AI分析所有图片
3. **质量筛选标准**：
   - 分辨率 ≥ 800x600
   - 主体清晰，非logo/icon
   - 构图完整
   - 视觉吸引力强

**界面展示**

封面图样式：
```
┌────────────────┐
│                │
│                │  图片比例：16:9 或 4:3
│   [封面图]     │  圆角：8px
│                │  阴影：subtle shadow
│                │  Hover：轻微放大效果
│                │
└────────────────┘
```

交互功能：
- 点击封面图：大图预览（Lightbox）
- 悬停显示：图片尺寸信息
- 可手动更换封面：从案例的其他图片中选择

#### 2.3.2 设计理念概括

**AI分析维度**

```
输入：封面图 + 案例标题 + 案例描述文本

AI Prompt 示例：
─────────────────────────────────────────
请分析这个设计案例，从以下维度进行专业评析：

1. 设计风格：识别主要风格（极简/扁平/拟物/新拟态等）
2. 色彩方案：分析主色调、配色特点和情绪表达
3. 布局特点：描述排版、视觉层次和留白处理
4. 设计元素：识别关键元素（插画/摄影/图标/字体）
5. 设计亮点：总结独特之处和创新点

输出要求：
- 200-300字
- 专业但易懂
- 结构化表达
─────────────────────────────────────────

输出：结构化的设计理念文本
```

**输出示例**

```

色方案，以柔和的灰蓝色（#8B9FAC）为主色调，营造出温柔优雅
的视觉氛围。

布局上运用经典的卡片式设计，大量留白（whitespace）增强了
内容的呼吸感和可读性。产品图片统一采用白底处理，保持视觉一
致性。

界面采用大尺寸的产品图（占屏幕40%）和简洁的文字说明，突出
商品本身，体现"少即是多"的设计理念。导航采用底部Tab栏设
计，符合移动端操作习惯。

设计亮点在于微妙的阴影处理和圆角细节，为扁平化设计增添了层
次感，整体风格清新、专业、易用。
```

## 3. 用户流程图

### 3.1 主流程：添加案例

```
开始
  │
  ├─→ [方式1] 点击工具栏"+ 插入案例"
  ├─→ [方式2] 输入斜杠命令 /case
  ├─→ [方式3] 快捷键 Ctrl+Shift+C
  │
  ↓
弹出URL输入框
  │
  ├─→ 粘贴链接（自动识别剪贴板）
  ├─→ 手动输入链接
  │
  ↓
验证URL格式
  │
  ├─→ [有效] 继续
  └─→ [无效] 显示错误提示 → 返回输入
  │
  ↓
创建空白卡片（骨架屏）
  │
  ↓
后台抓取内容
  │ (显示进度：0-30%)
  │
  ├─→ [成功] 获取标题、图片、描述
  └─→ [失败] 显示错误 → [重试] 或 [手动添加]
  │
  ↓
调用AI分析
  │ (显示进度：30-90%)
  │ ├─ 封面图提取
  │ ├─ 设计理念生成
  │ └─ 提示词生成
  │
  ├─→ [成功] 填充卡片内容
  └─→ [失败] 显示基础信息（降级处理）
  │
  ↓
渲染完整卡片
  │
  ├─→ 用户可编辑内容
  ├─→ 添加自定义标签
  ├─→ 移动到文件夹
  │
  ↓
自动保存
  │
  ↓
完成
```

### 4.3 配色方案

**主色调**
- 主色（Primary）：`#5B7FFF` 专业蓝
- 主色-浅（Light）：`#E8F0FF` 浅蓝背景
- 主色-深（Dark）：`#4A66CC` 深蓝（Hover状态）

**中性色**
- 文字-主要：`#1A1A1A` 深黑
- 文字-次要：`#666666` 中灰
- 文字-禁用：`#CCCCCC` 浅灰
- 边框：`#E0E0E0` 浅灰边框
- 背景：`#FFFFFF` 纯白
- 背景-浅：`#F7F8FA` 浅灰背景

**功能色**
- 成功（Success）：`#52C41A` 绿色
- 警告（Warning）：`#FAAD14` 橙色
- 错误（Error）：`#FF4D4F` 红色
- 信息（Info）：`#1890FF` 蓝色

### 4.4 字体和间距

**字体**
- 字体家族：系统默认字体栈
  ```
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI',
               Roboto, 'Helvetica Neue', Arial, sans-serif;
  ```
- 标题1（H1）：28px / Bold / Line-height: 1.3
- 标题2（H2）：24px / Bold / Line-height: 1.4
- 标题3（H3）：20px / SemiBold / Line-height: 1.4
- 正文：14px / Regular / Line-height: 1.6
- 小字：12px / Regular / Line-height: 1.5

**间距系统（8px基准）**
- XS：4px
- S：8px
- M：16px
- L：24px
- XL：32px
- XXL：48px

### 4.5 交互动画

**过渡效果**
- 颜色变化：`transition: color 0.2s ease`
- 背景变化：`transition: background 0.2s ease`
- 阴影变化：`transition: box-shadow 0.3s ease`
- 位移动画：`transition: transform 0.3s ease`

**加载动画**
- 骨架屏：闪烁效果（shimmer）
- 进度条：平滑过渡，0.5s更新一次
- Spinner：旋转动画，1s一圈

**反馈动画**
- 按钮点击：按下效果（scale: 0.98）
- 卡片悬停：上升效果（translateY: -4px）
- Toast通知：从上方滑入，3秒后淡出

## 6. 技术要求（简化版）

### 6.1 技术栈建议

**前端**：React/Next.js + Tailwind CSS + Slate.js（富文本编辑器）
**后端**：Node.js/Python + FastAPI/Express
**AI服务**：OpenAI GPT-4V 或 Claude API（多模态分析）
**爬虫**：Puppeteer（无头浏览器）
**数据库**：PostgreSQL（案例数据）+ Redis（缓存）
**存储**：云存储服务（阿里云OSS/AWS S3）

### 6.2 核心技术模块

**网页爬取模块**
- 使用Puppeteer渲染JS页面
- 针对主流设计网站（Dribbble、Behance等）编写适配器
- 提取：标题、作者、图片、描述、标签

**AI分析模块**
- 调用多模态LLM（GPT-4V/Claude）
- 输入：图片URL + 文本描述
- 输出：结构化JSON（封面、理念、提示词）
- 优化Prompt提高生成质量

**编辑器模块**
- 基于Slate.js或Tiptap构建
- 自定义节点类型（CaseCard Node）
- 支持拖拽、复制、删除等操作
